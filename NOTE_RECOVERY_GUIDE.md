# 笔记恢复指南

## 🚨 笔记丢失问题解决方案

### 问题诊断
您的笔记内容消失可能有以下几种原因：
1. **账户切换**：登录了不同的账户，笔记保存在其他账户中
2. **数据清理**：意外清理了应用数据或缓存
3. **存储问题**：本地存储空间不足或数据损坏
4. **应用更新**：应用更新过程中数据迁移失败

## 🔧 恢复方法

### 方法一：使用数据恢复工具（推荐）

#### 步骤1：打开数据恢复工具
1. 进入 **账户管理** 页面
2. 点击 **数据恢复工具** 选项
3. 系统会自动扫描可恢复的数据源

#### 步骤2：选择恢复源
系统会显示以下可恢复的数据源：
- **本地备份**：自动创建的备份数据
- **Dan账户**：保存到Dan账户的笔记
- **其他账户**：其他账户的笔记数据
- **临时数据**：应用临时保存的数据

#### 步骤3：预览和恢复
1. 点击 **预览** 查看数据内容
2. 确认无误后点击 **恢复**
3. 等待恢复完成

### 方法二：通过账户管理恢复

#### 步骤1：检查账户数据
1. 进入 **账户管理** 页面
2. 点击 **查看Dan账户** 查看是否有保存的笔记
3. 如果有数据，点击 **保存到Dan账户** 的相反操作

#### 步骤2：手动恢复
1. 点击 **恢复笔记** 按钮
2. 选择要恢复的数据源
3. 确认恢复操作

### 方法三：创建新备份

如果以上方法都无法恢复，建议：
1. 点击 **创建备份** 保存当前状态
2. 重新开始创建笔记
3. 定期使用备份功能

## 📋 恢复操作详细步骤

### 1. 进入数据恢复页面
```
账户管理 → 数据恢复工具
```

### 2. 查看当前状态
- 显示当前笔记数量
- 显示最后更新时间
- 确认是否真的丢失了数据

### 3. 选择恢复源
系统会显示所有可用的数据源：
- 💾 **本地备份** (X条笔记)
- 👤 **Dan账户** (X条笔记)  
- 📝 **临时数据** (X条笔记)

### 4. 预览数据
点击 **预览** 按钮查看：
- 笔记标题列表
- 创建时间
- 数据来源信息

### 5. 执行恢复
点击 **恢复** 按钮：
- 系统会提示确认
- 恢复过程会显示进度
- 完成后自动跳转到笔记页面

## ⚠️ 注意事项

### 恢复前须知
1. **数据覆盖**：恢复操作会覆盖当前的笔记数据
2. **备份建议**：恢复前建议先创建当前数据的备份
3. **数据完整性**：恢复的数据可能不是最新的

### 恢复后检查
1. 检查笔记数量是否正确
2. 验证笔记内容是否完整
3. 确认标签和分类是否正确
4. 测试知识星图功能是否正常

## 🛡️ 预防措施

### 定期备份
1. 进入账户管理页面
2. 点击 **创建备份** 按钮
3. 系统会自动保存当前所有笔记

### 账户管理
1. 使用 **保存到Dan账户** 功能
2. 将重要笔记保存到特定账户
3. 避免频繁切换账户

### 存储管理
1. 定期清理缓存
2. 监控存储空间使用情况
3. 避免存储空间不足

## 🔍 故障排除

### 问题1：没有找到可恢复的数据
**解决方案**：
- 检查是否在其他设备上登录过
- 确认是否使用过账户保存功能
- 联系技术支持

### 问题2：恢复后数据不完整
**解决方案**：
- 尝试从其他数据源恢复
- 检查是否有多个备份版本
- 手动补充缺失的笔记

### 问题3：恢复操作失败
**解决方案**：
- 检查存储空间是否充足
- 重启应用后重试
- 清除应用缓存后重试
